# 单表查询

## 查询的语句的执行顺序。

```text
查询的语句的执行顺序。

1.找到表:from

2.拿着where指定的约束条件，去文件/表中取出一条条记录

3.将取出的一条条记录进行分组group by，如果没有group by，则整体作为一组

4.将分组的结果进行having过滤

5.执行select

6.去重

7.将结果按条件排序：order by

8.限制结果的显示条数
```



```sql
select distinct 字段1,字段2,字段3 from 库.表 
	where 条件
	group by 分组条件
	having 过滤
	order by 排序字段
	limit n;
语法顺序这样写
执行顺序是 where   group by    having   distinct   order by  limit
```

##  分组查询  group by

```sql
#1、首先明确一点：分组发生在where之后，即分组是基于where之后得到的记录而进行的

#2、分组指的是：将所有记录按照某个相同字段进行归类，比如针对员工信息表的职位分组，或者按照性别进行分组等
    取每个部门的最高工资
    取每个部门的员工数
    取男人数和女人数

小窍门：‘每’这个字后面的字段，就是我们分组的依据
```

```sql
select * from employee group by post; 
# 会报错 需要先设置 set global sql_mode="ONLY_FULL_GROUP_BY"; 
# 这样只能去分组字段
select 这里要么写分组字段，要么就是聚合函数，group_concat(),其他都会报错 from  employee group by post;

```

## group_concat

分组后拿到每个组的下信息。

```sql
如按职位分组后，拿到每个组的名字
select post,group_concat(name) from employee group by post;
 
```

**强调：**

```text
如果我们用unique的字段作为分组的依据，则每一条记录自成一组，这种分组没有意义
unique就是唯一，以它分组没有意义。
多条记录之间的某个字段值相同，该字段通常用来作为分组的依据
如果不分组就按一个分。
```

## 聚合函数

```sql
max 	   最大值

```

使用

```sql
#每个职位有多少个员工  修改名字。

```

## having 过滤

### having和where的区别

```sql
#！！！执行优先级从高到低：where > group by > having 
#1. Where 发生在分组group by之前，因而Where中可以有任意字段，但是绝对不能使用聚合函数。

#2. Having发生在分组group by之后，因而Having中可以使用分组的字段，无法直接取到其他字段,可以使用聚合函数
```

### 案例

```sql
# 1. 查询各岗位内包含的员工个数小于2的岗位名、岗位内包含员工名字、个数
select post,group_concat(name),count(id) from employee group by post having max(id) < 2;
```

## order by 查询排序

从小到大，是降序
从大到小，是升序

```sql
#  通过salary不指定是升序
select * from employee order by salary;    
# asc 是升序
select * from employee order by salary asc;   
# desc 是降序
select * from employee order by salary desc;

# 按照年龄的降序来排，年龄相同就按照id的升序来排。
select * from employee order by age desc,id asc;

```


## 正则查询

模糊匹配(like) 中的_ 和% 太少了，用到正则

```sql
# regexp 后写正则表达式，正则的语法在sql中大部分一样。
select * from employee where name regexp '^jin';
select * from employee where name regexp '^jin.*(g|n)$';
```

## 连表操作

```sql
内连接：只取两张表的共同部分
select * from  book inner join author on book.press_id = author.id ;

左连接:在内连接的基础上保留左表的记录
select * from book left join autho on book.press_id = author.id ;

右连接：在内连接的基础上保留右表的记录
select * from book right join author on  book.press_id= author.id ;
全外连接 ：在内连接的基础上保留左右两表没有对应关系的记录
# 全外连接 语句长
select * from book left join author on  book.press_id = author.id
union
select * from author right join book on  book.press_id = author.id ;

```

## 多表查询的语法

```sql
# 语法是先去重，字段，

select distinct <select_list>          
from <left_table>
<join_type> join <right_table>
on <join_condition>
where <where_condition>
group by <group_by_list>
having <having_condition>
order by <order_by_condition>
limit <limit_number>
```
