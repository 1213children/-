# 表关系

两张表之间的关系
一对多
一对一
多对多

## 一对多

```text
如何找到表与表的关系
就是在左表看看有没有多条记录对应右表的一条记录。
然后再从右表看看有没有多条记录对应左表的一条记录
只要有一个表找到了。
就是一对多的关系。
```

上一个案例就是一对多的关系。

```sql
# 建立被关联表
create table press(
id int primary key auto_increment,
name varchar(20)
);
# 再建立关联表
create table book(
id int primary key auto_increment,
name varchar(20),
press_id int not null,
foreign key(press_id) references press(id)
on delete cascade
on update cascade
);
# 使关联表的press_id关联上press表的id
```

## 多对多

多对多需要三个表

让对应关系放到第三个表中。

```tex
多对多：一个作者可以写多本书，一本书也可以有多个作者，双向的一对多，即多对多
找的时候还是
就是在左表看看有没有多条记录对应右表的多条记录。
然后再从右表看看有没有多条记录对应左表的多条记录
只要一个表找到了。
就是d对多的关系。
```



```sql
# 建第一个表存放书名
create table book(
id int primary key auto_increment,
name varchar(20),
press_id int not null);
# 在建一个表存放作者
create table author(
id int primary key auto_increment,
name varchar(20)
);
#  有多个作者写同一本书，也有多本书是有一个作者写的，这就是多对多。

create table author2book(
id int not null unique auto_increment,
author_id int not null,
book_id int not null,
foreign key(author_id) references author(id)
on delete cascade
on update cascade,
foreign key(book_id) references book(id)
on delete cascade
on update cascade,
primary key(author_id,book_id)
);
# 创建第三表，存放作者和书的对应关系。
# 使author_id关联author表的id字段
# 使book_id关联book表的id字段


# 插入数据
insert into author(name) values('沈泽昊'),('曹璐'),('董瑞'),('张辛');

insert into book(name,press_id) values("九阳神功",1),("独孤九剑",2),("降龙十巴掌",3),("葵花宝典",4),("九阴真经",5),("九阴真经",6);
insert into author2book(author_id,book_id) values
(1,1),
(1,2),
(1,3),
(1,4),
(1,5),
(1,6),
(2,1),
(2,6),
(3,4),
(3,5),
(3,6),
(4,1)
;
+----+-----------+---------+
| id | author_id | book_id |
+----+-----------+---------+
|  1 |         1 |       1 |
|  2 |         1 |       2 |
|  3 |         1 |       3 |
|  4 |         1 |       4 |
|  5 |         1 |       5 |
|  6 |         1 |       6 |
| 15 |         2 |       1 |
| 16 |         2 |       6 |
| 17 |         3 |       4 |
| 18 |         3 |       5 |
| 19 |         3 |       6 |
| 20 |         4 |       1 |
+----+-----------+---------+
# 这样就清晰可见了
比如作者1写过的书有1，2，3，4，5，6
   作者2写过的书有1,6
所有他们同时写过书1，6
```

## 一对一

```text
一对一：在辅导班一个学生是一个客户，一个客户有可能变成一个u，即一对一的关系。
就是在左表看看有没有一条记录对应右表的一条记录。
然后再从右表看看有没有一条记录对应左表的一条记录
只要一个表找到了。
就是一对多的关系。
```

```sql
create table customer(
id int primary key auto_increment,
name varchar(20) not null,
qq varchar(10) not null,
phone char(16) not null
);


create table student(
id int primary key auto_increment,
class_name varchar(20) not null,
customer_id int unique, #该字段一定要是唯一的
foreign key(customer_id) references customer(id) #外键的字段一定要保证unique
on delete cascade
on update cascade
);
```
